# 技能：定时任务

在每天指定时间自动执行周期性任务。

## 所需 Allowlist 命令

无额外命令（使用核心命令中已包含的 `Shell(sleep)` 和 `Shell(curl)`）。

## 工作原理

在每次轮询循环中，检查用户时区的当前时间，与 `<AGENT_DIR>/memory.md` 中的定时任务进行匹配。

### 获取当前时间

```bash
TZ=<用户时区> date '+%H:%M'
```

### 匹配规则

- 当前时间在目标时间的 ±5 分钟范围内即匹配（例如目标 08:00，匹配 07:55~08:05）
- 每个定时任务每天**只执行一次**
- 维护已执行任务的内部记录，避免重复
- 日期变更时重置记录

### 任务存储

定时任务存储在 `<AGENT_DIR>/memory.md` 的 **「Skill: Scheduled Tasks」** 章节下。用户可以通过对话添加、修改或删除任务（例如"每天早上8点提醒我吃药"、"取消8点的提醒"）。

### 示例流程

```
轮询循环:
  1. sleep 5
  2. 检查时间：TZ=Asia/Shanghai date '+%H:%M' → 08:02
  3. memory.md 中有："每天早上8点提醒吃药"
  4. 08:02 在 08:00 的 ±5 分钟内，且今天未执行
  5. → 通过渠道发送消息："早上好！该吃药啦 💊"
  6. 标记"提醒吃药"已执行
  7. 继续读取 watch 输出并处理消息
```

### 外部数据任务

当任务涉及查询外部数据（天气、股价等）时，使用 curl 查询公开 API：

- 天气：`curl -s "wttr.in/<城市>?format=..."`
- 股价：任何公开的股票 API

## 引导设置

引导设置时，执行以下步骤：

1. **检测时区**：运行 `readlink /etc/localtime | sed 's|.*/zoneinfo/||'` 获取 IANA 时区（如 `Asia/Shanghai`），保存到 `<AGENT_DIR>/memory.md` 的 **「Skill: Scheduled Tasks」** 章节。如果检测失败，询问用户住在哪里并推断时区。
2. **询问用户**（通过渠道）：
   > "你想要设置每日定时提醒吗？比如：'每天早上8点提醒我吃药'、'每天早上7点报天气'。以后随时可以添加。不需要的话回复'跳过'。"
